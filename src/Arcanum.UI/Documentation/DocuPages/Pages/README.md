# Arcanum Markdown Guide

Welcome to the Arcanum documentation engine. This engine supports standard Markdown along with several powerful custom extensions specifically built for the Arcanum UI.

This page serves as both a cheat sheet for writing documentation and a live test page for the renderer.

---

## Navigation and Linking

Arcanum extends standard Markdown links to allow deep integration with the application UI, commands, and other documentation files.

### 1. Internal Document Navigation
You can jump to any heading in the current document. The ID is automatically generated by converting the heading text to lowercase and replacing spaces with hyphens.
* **Syntax:** `[Jump to Alerts](#alerts)`
* **Example:** [Jump to Alerts](#alerts)

### 2. Custom Anchors
If you want to link to a specific paragraph instead of a heading, define a custom anchor using `{#your-id}` at the end of the line.
* **Syntax:** `[Go to Secret](#secret-anchor)`
* **Example:** [Go to Secret](#secret-anchor)

### 3. Cross-Document References
To link to the documentation of another feature, use the `id:` prefix followed by the feature's ID. You can also append `#` to jump to a specific section in that file.
* **Syntax:** `[Editor Preferences](id:editor.map)`
* **Syntax:** `[Map Tests](id:editor.map#tests)`
* **Example:** [Open Editor Map](id:editor.map)

### 4. Executing App Commands
You can trigger WPF commands directly from the documentation using the `cmd:` prefix.
* **Syntax:** `[Open Queastor](cmd:editor.open_queastor)`
* **Example:** [Open Queastor](cmd:editor.open_queastor)

### 5. Inline Tooltips
For quick definitions without navigating away, use the `tooltip:` prefix.
* **Syntax:** `[Hover me](tt: "This text pops up!")`
* **Example:** [Hover me](tt: "This text pops up!")

### 6. Standard Web Links
Standard HTTP links will open in the user's default web browser.
* **Syntax:** `[GitHub](https://github.com/The-Arcanum-Project/Arcanum)`
* **Example:** [GitHub](https://github.com/The-Arcanum-Project/Arcanum)

## Images and Media

Arcanum supports generic attributes to control image alignment, sizing, and scaling behavior.
* **Available Alignments:** `.left`, `.center`, `.right`, `.stretch`
* **Available Scales:** `None`, `Fill`, `Uniform`, `UniformToFill`

**Centered with fixed width:**
`![Cat](cat.png){.center width=200}`
![Cat](cat.png){.center width=200}

**Right-aligned with fixed height and UniformToFill:**
`![Cat](cat.png){.right height=150 scale=UniformToFill}`
![Cat](cat.png){.right height=150 scale=UniformToFill}

**Animated Gifs:**
Standard syntax works seamlessly with animated gifs.
`![Cat Gif](cat.gif)`
![Cat Gif](cat.gif)

## Reusable Snippets

Arcanum includes a preprocessor that allows you to inject reusable blocks of text, warnings, or standard links across multiple files. This ensures consistency and makes updating common text trivial.

### Inserting a Snippet
Use the double-brace syntax with the `snippet:` prefix, followed by the ID of the snippet.

* **Syntax:** `{{snippet:danger_zone}}`

## Code formatting

You can format inline code using single backticks: `var x = 10;`.
For larger code blocks, use triple backticks and specify the language for syntax highlighting.

```csharp
public void HelloWorld()
{
    Console.WriteLine("Code blocks are fully supported!");
}
```
## Alerts

Arcanum supports GitHub-style alert blocks to draw attention to crucial information.

> [!NOTE]
> Notes highlight information that users should take into account, even when skimming.

> [!TIP]
> Tips provide optional information to help a user be more successful or efficient.

> [!IMPORTANT]
> Important blocks contain crucial information necessary for users to succeed.

> [!WARNING]
> Warnings indicate critical content demanding immediate user attention due to potential risks.

> [!CAUTION]
> Caution blocks warn of negative potential consequences of an action, such as data loss.

You can organize complex information into tabbed interfaces using the :::: tabs and ::: tab Name syntax. This is great for showing multiple programming languages or alternative methods side-by-side.

```csharp
var x = 10;
Console.WriteLine(x);
```

::: tab VB

```csharp
Dim x = 10

```

Contextual Sections:

You can restrict parts of the documentation to only show up in specific contexts (like Debug mode or Advanced mode) using section dividers.

```text
---section/:Debug---
Developer Note: This section is explicitly marked as Debug. It will only be rendered and visible if the documentation viewer is instantiated in a developer/debug context or an advanced tech-popup.
```
